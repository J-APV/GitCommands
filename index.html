Git Commands

Basic Git Commands

In order to push something to Github, your code will have to go through 3 stages

  1. Adding your new/updated files into the "staging" area:
    2. git add . (Which means add the current directory to the staging area)
    OR alternatively, if you want to only add one file to github, you can just add that specific file
    git add HelloWorld.txt
  2. Add your "staged" files include a small description of what feature you are pushing to commit your changes:
    3. git commit -m "Add commit message here"
  3. Finally, you upload your commited work onto github:
    4. git push origin main

Note:
You can do git pull to get the most recent version of the Github repo.
Make sure to always update your codebase before coding away :D

How to use Git when working on teams?

These 3 commands are enough if you're working on a project alone. However, when working in a team, you have to worry about having the most recent version of the code, while also making sure your changes aren't conflicting with others' code or vice versa.

For this reason, git has a feature called "branches". It's like creating a seperate timeline from the main timeline. Your branch will copy the recent code and you will be able to make changes to it without worrying about interfering with other's making changes. You will work in your branch until your feature is completely finished. Once finished, you will retrieve the most recent version of the Github repository and merge your branch onto the main repository.

These are the commands you have to do:
  1. git branch [branch-name] *It is recommended to make your branch name like this: [nameinitials_feature] ex: japv_shoppingcart
  2. git checkout [branch-name] *This switches you from the "main" branch onto the branch you just made!

    3. Alternatively... you can do those 2 steps in one line like this:

    1.) git checkout -b [branch-name] *The -b means "git, please create a new branch"

  3. git fetch *This just gets the most recent version of the code, similar to pull but it won't make changes to your files just yet
  4. git push -u origin [branch-name] *This pushes the branch onto Github so your teammates can also join your branch if you need to work together

    5. You should add and commit your code often! This will save your code on github just in case anything bad happens! If something happens to the files your working on, you can always go back to them on your github branch :D

  5. git add .
  6. git commit -m "description here"
    7. Once you're finished with your feature/task and you're ready to merge:
  7. git fetch *Making sure your main repo is the most recent
  8. git checkout main
  9. git merge [branch-name]*Merges your branch onto main
  10. git push origin main *Pushes your version of the code(merged version) onto github
    11. Now, we should delete your branches that exist remotely and also locally in your computer
  11. git push origin :[branch-name] *The semi-colon is important and tells git to "delete". (This deletes branch remotely)
  12. git branch -D [branch-name] *Deletes the branch from your local repository

Some extra notes:

If 2 or more people are working with similar files, it’s important to switch to their branch instead of creating your own branch:

  1. git fetch *Gets any branches that may exist in the Github
  2. git branch -a *To see all branches, including your branches and remote branches
  3. git switch [desired-remote-branch-name] *Switches to the branch X person is working on

Vocabulary

  • Pushing on git (means uploading your code onto the github repository and saving it there)
  • Pull on git(means getting the most recent version from github and changes your files to recent)
  • Fetch on git(means getting the most recent version but it doesn’t change your files until u merge)
  • Forking (means grabbing someone else’s REPO on github but it’s saved under you. So when u push, it will save under the repo you own, instead of the person you got it from)
  • Branch (Alternate repository so you can create new features without affecting the working code)
  • Origin - Refers to github remote or “original” repo
  • Main - where the working code goes.
  • The command -a means “all”
  • The command -b means “create a new branch” ex: git checkout -b [branch-name]
  • The command -u means “upstream branch” and it allows git to track the branch you created and links it to a branch on github. So now you can do git pull when working on a branch without specifying what branch you want to pull from.
